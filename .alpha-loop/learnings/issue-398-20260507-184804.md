## Patterns
- **Persist-on-write + compute-on-read fallback**: When fixing a value-derivation bug, persist the correct value going forward AND compute it from source columns at read-time for legacy rows. Avoids forcing backfill as a prerequisite.
- **Single-call invariant tests**: Test like `test_completed_at_passed_once_to_update_status` locks in the contract that `completed_at` is computed once and reused, preventing future drift between persisted and derived timestamps.

## Anti-Patterns
- Measuring "real duration" inside a tick loop that runs in fast-forward simulated time — the loop's wall-clock duration has no relationship to the user-visible run span. Always derive durations from boundary timestamps (started_at, completed_at), never from intermediate loop accumulators.
