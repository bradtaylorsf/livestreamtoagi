## What Worked
- Reused existing `POST /api/admin/simulations` by extending it with optional `seed_file` + `max_cost` fields rather than creating a parallel endpoint, keeping a single launch path.
- Path-traversal guard on `seed_file` (resolve + `relative_to(scenarios_dir)`) blocks absolute paths and `../` escapes; covered by three dedicated tests.
- Pre-creating the simulation row with `status="running"` before spawning the subprocess gives the redirect target a valid row immediately, avoiding a race with the orchestrator attach path.

## What Failed
- Nothing — all 1944 backend + 335 frontend + 140 website tests passed on first run.

## Patterns
- Extracting `_project_root()` as a tiny helper makes path-dependent route logic monkey-patchable in tests without `__file__` gymnastics.
- Best-effort YAML description parsing (read leading `#` comment block, stop at first blank line after content) avoids a full YAML parse just to surface a one-line description in a dropdown.
- Pydantic `Field(ge=0, le=10)` on `max_cost` pushes validation to the framework boundary, eliminating manual range checks in the handler.

## Anti-Patterns
- Don't add polling on a detail page when the issue explicitly scopes it as a stop-gap; reviewer correctly flagged it as `info`/unfixed rather than reworking scope.
- Don't silently relocate an endpoint path (`/api/scenarios` → `/api/admin/scenarios`) without noting the deviation; reviewer caught the spec mismatch but accepted the rationale (admin-gated action).
