## What Failed

## Patterns
- When relaxing a filter default, branch the query rather than passing a sentinel — keeps both code paths explicit and testable.
- Assert the SQL string itself (e.g., `"simulation_id" not in query`) to lock in scope behavior, not just the response shape.
- Expose foreign-key IDs (like `simulation_id`) on list response models when the UI may need to group or deep-link.

## Anti-Patterns
- Defaulting list endpoints to a single tenant/scope ID (`LIVE_SIMULATION_ID`) silently hides historical data — prefer unscoped-by-default with explicit filtering.
- Relying on `.gitignore` directory entries to cover symlinks of the same name; symlinks need explicit handling or pre-commit checks.
