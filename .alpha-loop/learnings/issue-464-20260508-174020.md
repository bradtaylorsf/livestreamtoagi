## Patterns
- Hydration-order bugs where `useSearchParams()` returns null on initial render are best fixed with a lazy `useState` initializer that reads `window.location.search` directly (SSR-guarded with `typeof window !== "undefined"`), keeping the `useEffect` sync for subsequent client navigation.

## Anti-Patterns
- Tests assert source structure via regex rather than rendering the component (no jsdom/RTL configured in `website/`). Matches existing convention in `clips-page.test.ts`, but a real React Testing Library test would more directly verify cold-load behavior and would survive a non-cosmetic refactor.
- Deriving initial state purely from `useSearchParams()` (which returns null/empty on first client render after SSR) — silently breaks deep links and only works after a `router.push()` event.
