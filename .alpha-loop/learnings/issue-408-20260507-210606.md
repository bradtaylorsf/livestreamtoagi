## Patterns
- Stats endpoint pattern: frontend reads counts from a single `/api/stats` endpoint rather than hard-coding, keeping UI in sync with DB state automatically.
- Persisted simulation filter via sessionStorage hydrated into initial fetch calls (not just filter UI inputs) — must include the persisted ID in the first request, not only on subsequent re-triggers.
- Path-traversal validation pattern for user-supplied file paths: resolve candidate against allowed dir, then `relative_to()` to assert containment before accepting.

## Anti-Patterns
- Bundling unrelated work into a single issue's diff makes review harder and obscures the actual fix; #408 was a 2-line stat fetch but the diff touches admin routes and route restructuring.
- Initializing `useState` from `window`/sessionStorage on first render causes SSR hydration mismatches — initialize to `null` and sync in `useEffect` instead.
- Declaring an AC as "all endpoints" but implementing only a subset creates AC drift; either narrow the AC or implement uniformly.
