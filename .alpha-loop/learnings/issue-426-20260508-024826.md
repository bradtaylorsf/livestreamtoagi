## What Worked
- All 2092 backend pytest tests, 335 frontend tests, and 240 website tests passed on first run with zero test-fix retries.
- Reviewer caught two real UX bugs (blank video thumbnails, broken scoped breadcrumb) and fixed them in-loop rather than deferring.
- Issue scope was respected: home page restructured into Hero/Featured/Recently completed/Currently running, Cast moved to `/agents`, blog kept in footer area.

## What Failed
- Initial implementation used `<video poster={sim.video_url}>` where `video_url` is an MP4, not an image — poster attribute silently rendered nothing.
- Scoped agent detail breadcrumb hardcoded `/agents` instead of preserving sim scope via `/simulations/[id]/agents`.
- Scope creep: PR also added a public user auth system (magic-link, JWT cookies, `/api/auth/*`) and simulation submit endpoint that are not part of issue #426's acceptance criteria.

## Patterns
- For video thumbnails without a separate poster image, use `<video src={url} preload="metadata">` to let the browser render the first frame.
- When building scoped routes (e.g. `/simulations/[id]/...`), thread an `isRouteScoped` flag through breadcrumbs/links so navigation preserves context.
- Concurrent test runner (`concurrently -g -n pytest,frontend,website`) keeps a 2100+ test suite under 75s wall time.

## Anti-Patterns
- Don't pass a video URL to an image-typed attribute (`poster`, `<img src>`) — verify the attribute's expected MIME type.
- Don't hardcode top-level paths inside components that may render under a scoped route; derive the prefix from route context.
- Don't bundle unrelated infrastructure (auth, submit API) into a UI redesign issue — it inflates review surface and couples unrelated risks.
