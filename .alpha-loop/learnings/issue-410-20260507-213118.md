- Skeleton loaders replaced text "Loading..." across major dashboard pages (simulations list, agent tabs, conversation lists, eval pages) matching issue AC.

## What Failed

## Patterns
- Pair `SkeletonBlock`/`SkeletonGrid`/`SkeletonCardList` with `useDelayedFlag(true)` for the "skeleton if slow, nothing if fast" UX pattern.
- When an issue lists target areas (e.g., "eval pages", "agent tabs"), grep for shared loading components used by those areas before declaring done — one shared spinner can cover many surfaces.

## Anti-Patterns
- Don't treat "list, page, detail" as the only loading surfaces — shared sub-components like `TabSpinner` are easy to miss but cover many user-facing tabs.
- Don't ship skeletons without minimum-delay gating; instant loads will flash the placeholder and look worse than text.
