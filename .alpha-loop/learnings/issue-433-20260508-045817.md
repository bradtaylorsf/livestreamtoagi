## Patterns
- When migrating a "separate concept" table to be backed by another entity, add a boolean flag (`shared_as_challenge`) on the source entity rather than a hard schema migration — preserves legacy rows via an `include_legacy` query param.
- Filter-with-escape-hatch pattern on list endpoints: default filters to the new behavior (shared-only), but accept an explicit override flag for backward compatibility / debugging.
- Public auth via magic-link: check signing secret BEFORE consuming the single-use token, otherwise misconfigured deploys silently burn user tokens.

## Anti-Patterns
- Implementing a backend "share" endpoint before any UI surface to invoke it creates dead code paths visible only via API. If the dependency (auth UX) isn't ready, leave a TODO marker on the simulation detail page so the wiring isn't forgotten.
- Exposing email local-part as `submitter_display_name` via `split_part(email, '@', 1)` is convenient but leaks PII; a dedicated `display_name` column would be cleaner from day one.
- Resetting a `cancelledRef` flag at the top of a new effect (SimulationWall tab switch) without aborting in-flight requests creates a visible flash of stale data; use AbortController or a tab-id token.
