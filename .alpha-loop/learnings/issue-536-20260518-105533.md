## Patterns
- Reuse an existing contract test's resolution helper (`MODEL_NAME_ALIASES` → `MODEL_REGISTRY`) as a runtime drift guard in new generators, so config and `core/llm_client.py` cannot silently diverge.

## Anti-Patterns
- Adding `package.json` scripts without updating the corresponding doc — sibling scripts (E3-3) set the documentation precedent that should have been followed in the same change.

## Suggested Skill Updates
- `docs-sync`: Add an explicit trigger for new `package.json` scripts — when a `verify:*`/`mc:*` script is added, require a matching entry in the relevant `docs/` runbook (and a "Where this is recorded" pointer) if sibling scripts are documented there.
