## Patterns
- Default `simulation_id` query param to `None` and pass `None` directly to repos, letting the repo layer handle "all sims" aggregation rather than substituting `LIVE_SIMULATION_ID` at the route layer.
- Journal endpoint uses an explicit if/else SQL split when `simulation_id` is None vs set, since asyncpg parameter binding doesn't cleanly support conditional WHERE clauses.
- Helper function with keyword-only args (`*, db, services, simulation_id`) makes call sites self-documenting when assembling a multi-source response model.

## Anti-Patterns
- Hard-coding `LIVE_SIMULATION_ID` as a default in route handlers — silently scopes data to one simulation and breaks any non-live context. Prefer optional params with `None` meaning "unscoped".
- Duplicating cost-aggregation/profile-assembly logic between list and detail endpoints — extract to a helper before the third copy appears.
