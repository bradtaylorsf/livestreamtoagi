## What Worked
- All 1998 backend tests + 335 frontend + 211 website tests passed on first run with zero test-fix retries.
- Migration + repo + admin route + orchestrator auto-population of baseline outcomes implemented cleanly with new test coverage (`test_baseline_outcomes_written_on_finalize`, `test_auto_draft_learnings_*`, `test_update_research_fields_*`).
- Reviewer caught and fixed a real gap: public `GET /api/simulations/{id}` was missing the new research fields despite being named in the verification step.

## What Failed
- Implementer scoped the work to admin routes only and missed the public GET endpoint, even though the issue's Verification section explicitly referenced `GET /api/simulations/{id}`.
- Acceptance criterion called for `POST /api/simulations` (public) accepting hypothesis; no public POST exists and none was added — gap left unfixed.
- CLI `--hypothesis` is silently dropped when combined with `--sim-id` (existing-row branch never calls `update_research_fields`).
- Diff shows unrelated factions/#419 changes (speaker_selector, conversation_engine, context_assembly, scenario YAML) bundled into this PR — scope creep beyond issue #418.

## Patterns
- When acceptance criteria list multiple endpoints (admin + public), grep for both route prefixes before declaring done; verification steps name the canonical endpoint to test against.
- Auto-populated baseline fields at finalize-time (outcomes from existing eval scores/counts) keep the data model useful even when users don't manually fill it in.
- Optional LLM-drafted learnings gated by config flag (off by default) is the right shape for cost-sensitive enrichment features.

## Anti-Patterns
- Reading acceptance criteria as admin-only when public endpoints are also listed — both `POST /api/simulations` and `POST /api/admin/simulations` were called out and only the admin one was addressed.
- Bundling unrelated issue work (#419 factions) into a #418 PR — makes review harder and muddies the change history.
- Skipping the existing-sim-id branch when adding a new creation parameter — `--hypothesis` only works on the new-row path.

## Suggested Skill Updates
- `implementation-planning`: add a checklist item "grep for ALL endpoints named in acceptance criteria + verification steps (public AND admin) before marking complete."
- `code-review`: add check "diff scope matches issue number — flag changes touching files unrelated to the stated issue."
