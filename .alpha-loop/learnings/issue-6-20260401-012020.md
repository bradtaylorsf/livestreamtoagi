## Patterns
- **Config-as-directory pattern**: Each agent gets its own directory (`agents/{id}/`) with typed YAML config, markdown prompt, and YAML behaviors — clean separation of structured data from prose
- **Spec-driven validation**: Validating model names against the existing `MODEL_REGISTRY` in `llm_client.py` prevents config/code drift at startup rather than at first LLM call
- **Graceful degradation**: Skipping invalid agents with logging (rather than crashing) and falling back from Redis to in-memory keeps the system resilient during development

## Anti-Patterns
- **Undocumented fields**: `eavesdrop_tendency` exists in configs but isn't in CLAUDE.md agent roster — fields should be documented in the spec before being added to configs to avoid confusion
- **Template placeholders without injection**: System prompts contain `{injected_from_sentinel}`, `{injected_from_twitch}` placeholders with no runtime injection yet — these will render as literal strings if prompts are used before the injection system is built
