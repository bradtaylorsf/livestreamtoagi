## Patterns
- YAML metadata extraction with layered fallback (explicit `meta:` block → leading comment block → phase aggregation) keeps backfill optional while letting authors override.
- Pure-helper unit tests (`buildPickerLabel`, `filterSimulations`) for picker components — fast, deterministic, matches existing repo convention.
- Issue-text verification numbers (e.g., "19 cards") should be cross-checked against actual repo state during implementation rather than treated as authoritative.

## Anti-Patterns
- Don't trust verification counts in issue text without verifying — issue said 19 scenarios, repo has 18.
- Avoid linking CTAs to routes that don't exist yet without flagging the forward dependency in code comments or PR description (here it was a known E2-13 dep, but easy to miss in review).
