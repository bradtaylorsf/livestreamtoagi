## Patterns
- Raw SQL migrations with `CREATE TABLE IF NOT EXISTS` and `ON CONFLICT DO NOTHING` provide simple idempotency without ORM overhead
- Separate up/down SQL files with a `schema_migrations` tracking table is a clean lightweight alternative to Alembic for fixed schemas

## Anti-Patterns
- Not verifying Docker service health before running integration tests — likely caused all 3 retry cycles
- Test output from failed attempts was not captured, making it impossible to diagnose what specifically broke across retries

## Suggested Skill Updates
- Add a pre-step to integration test skills: always run `scripts/check-services.sh` and wait for healthy services before executing database-dependent tests
- Capture and preserve stderr/stdout from failed test attempts so retry root causes are diagnosable
