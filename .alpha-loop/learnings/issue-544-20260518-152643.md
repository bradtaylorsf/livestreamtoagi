## Patterns
- Fail-closed resilience: distinct typed error codes per failure mode (`bridge_unreachable` vs `bridge_overloaded`), both routing the bot to safe-idle without ever throwing — verifiable by asserting "never crashes" plus the specific code.

## Anti-Patterns
- Verifying a behavioral requirement (bounded concurrency / shedding) with a source-grep instead of exercising the behavior under a concurrent burst — a static check cannot prove the bound holds at runtime.

## Suggested Skill Updates
- `testing-patterns` / `code-review`: add a rule that scope items naming a runtime behavior (concurrency limits, backpressure, rate caps) require a behavioral test exercising the limit, and that a string-grep alone is insufficient evidence — this is exactly the gap the reviewer had to backfill here.
