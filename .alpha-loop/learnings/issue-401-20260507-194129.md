## What Worked
- Implementation landed on first pass with all 1944 backend tests, 335 frontend tests, and 146 website tests passing — no test-fix retries needed.
- Reviewer caught a critical polling bug (missing `status` field on `PublicEvalRunDetail`) before it shipped and added a backend regression test for the fix.
- Path traversal guard on `seed_file` (resolve + `relative_to(scenarios_dir)`) was implemented correctly and tested with absolute-path and outside-dir cases.

## What Failed
- Initial `PublicEvalRunDetail` Pydantic model omitted the `status` field even though the frontend polled on it — silent contract mismatch between TS interface and backend response would have caused 10-minute polling timeouts on every successful run.

## Patterns
- When adding polling UI, verify the polled field actually exists on the response model (both Pydantic schema and TS interface) — don't trust the cast.
- Subprocess-launched scenario runs: validate user-supplied paths via `Path.resolve().relative_to(allowed_dir)` rather than string prefix checks.
- Extract helpers like `_project_root()` and `_build_agent_profile()` to keep route handlers thin and unit-testable without `__file__` monkey-patching.

## Anti-Patterns
- TypeScript `as` casts on API response shapes hide missing-field bugs; the unsafe cast on `detail.status` masked the contract mismatch until review.
- Route handlers that ignore path params (e.g., `sim_id` in the snapshot GET) create misleading URLs even when not exploitable — either use the param or drop it.
