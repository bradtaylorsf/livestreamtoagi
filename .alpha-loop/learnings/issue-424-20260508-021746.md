## What Worked
- Email notification flow shipped with both success and failure paths, opt-out via unsubscribe token, and provider mocking in tests (test_simulation_complete_notification.py) — all 2089 backend tests pass on first run.
- Magic-link auth scaffolding (core/auth/) reused for unsubscribe token lookup, keeping the user-identity surface consistent across endpoints.
- HTML + plaintext templating with embedded video thumbnail when available, falling back gracefully when video pipeline hasn't completed.

## What Failed
- Nothing blocking; three INFO-level integration gaps flagged by review (video link never populated at email time, missing /simulations/{id}/replay page, Playwright not in deps) — all spec-compliant or tracked elsewhere.

## Patterns
- Fail-closed on email delivery (503 + log, no false success) when provider misconfigured — prevents silent UX breakage.
- Check signing-secret config BEFORE consuming single-use tokens, so a misconfigured deploy doesn't burn user tokens on every click.
- Backfill missing per-user tokens lazily on first send rather than requiring a migration backfill.

## Anti-Patterns
- Detaching the video render as a subprocess and sending the completion email immediately means the "video link" branch is effectively dead in production — fire-and-forget + sequential notify creates a race the spec accepts but users won't.
- Lazy-importing a runtime dependency (Playwright) without declaring it in pyproject means unit tests pass while production renders fail with import errors.
