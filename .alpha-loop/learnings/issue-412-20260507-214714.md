## Patterns
- When replacing a "coming soon" placeholder, group structured data by type (room, decoration, etc.) for readable list/table rendering before investing in richer visualizations.
- Empty-state copy that explains *when* data will appear (vs. just "no data") is more helpful for bootstrap-phase features.
- Surfacing related signals (world_events) alongside primary data (chunks) gives the page substance even when chunk count is low.

## Anti-Patterns
- Shipping placeholder pages ("coming soon") when the backend data already exists — wastes the data and frustrates QA walkthroughs.
- Marking agents as "offline" by default on a page that has no live-status wiring — misleading UI state.
