## What Worked
- All tests passed on first run (1944 backend, 335 frontend, 146 website) with zero test-fix retries.
- Reviewer caught a critical polling bug where `PublicEvalRunDetail` lacked a `status` field, causing silent 10-min timeouts on completed evals; fixed with backend model update plus regression test.
- Path traversal protection on `seed_file` and snapshot filename inputs was implemented correctly using `resolve()` + `relative_to()` checks.

## What Failed
- Diff scope drifted well beyond issue #402: changes touched simulation creation (seed_file launcher), scenario listing endpoint, agent profile refactor, and `PublicEvalRunDetail.status` — none required by the snapshot-detail-view issue.
- Clone-from-snapshot button reuses the existing clone endpoint, which exports current live state rather than the specific snapshot file the user is viewing — literal AC met but semantic intent missed for older snapshots.
- Backend snapshot detail route accepts a `sim_id` path param but ignores it, allowing any admin to fetch any snapshot regardless of URL sim id (tidiness issue, not a security bug).

## Patterns
- When adding a Pydantic response model field that the frontend will poll on, verify the field is populated in the route handler AND exposed in the TS interface — missing either causes silent `undefined` checks.
- For filesystem inputs from API requests, always combine `resolve()` with `relative_to(allowed_dir)` to reject path traversal and absolute paths in one check.
- Adding a regression test alongside a polling/contract bug fix prevents future re-introduction when models are refactored.

## Anti-Patterns
- Bundling unrelated features (simulation launcher, scenario listing, agent profile refactor) into a single issue's PR — makes review harder and inflates blast radius.
- Reusing a "clone simulation" endpoint for "clone from snapshot" when the semantics differ (live state vs. frozen snapshot state) — meets literal AC but creates user-visible inconsistency.
- Accepting path parameters in routes and then ignoring them — invites confusion and authorization-shaped bugs even when not currently exploitable.
