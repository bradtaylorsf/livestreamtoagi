## Patterns
- Per-turn time-series persistence: bulk-write one row per active participant inside the existing turn-completion hook, keyed by (simulation_id, agent_id, conversation_id, turn_number).
- API response shape `{agent_id: [{t, energy, ...}]}` groups at read time via repo method, keeping write path simple while supporting per-agent filtering via query param.
- Pair the perf-sensitive endpoint with a synthetic-row test that asserts a wall-clock budget — catches regressions without needing a real simulation.

## Anti-Patterns
- Writing the same conversation-level value to per-agent rows: schema supports per-agent variation but the engine collapses it. Acceptable as a stub but document the limitation in the writer (done) so future per-agent energy work knows where to plug in.
- Unbounded log tables: `agent_energy_log` was excluded from retention sweep without a documented rationale. New time-series tables should explicitly opt in or out of cleanup with a comment.
- Returning a superset of the spec'd response shape (`{t, energy, turn, conversation_id}` vs spec `{t, energy}`) without flagging — harmless here but spec drift adds up.
