## Patterns
- URL-first context providers (read URL → fall back to session store) cleanly support both scoped and aggregate views without breaking old links.
- Pairing a new context provider with at least one scoped-route test catches regressions early; the test trio (provider unit + store + route integration) is a reusable template.
- Reviewer-as-second-pass caught semantic issues (poster vs src, breadcrumb scope) that type checks and tests cannot detect.

## Anti-Patterns
- Using `<video poster={url}>` with an MP4 URL — `poster` requires an image; use `<video src={url} preload="metadata">` to get a first-frame thumbnail.
- Hardcoding top-level paths (`/agents`, `/conversations`) inside components that may render under a scoped route — always check `isRouteScoped` and prefix with `/simulations/[id]` when applicable.
