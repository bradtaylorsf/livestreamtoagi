## What Failed
- One critical wiring gap (caught and fixed in review, not by tests): the contract `skipif`s when no Mindcraft clone is present, so it only ever executes during an E3-6 re-base — yet the re-base runbook never instructed running it, leaving E3-7's exact named regression vector open in practice despite a correct test.

## Patterns
- For a conditional issue whose trigger is unmet, satisfy the acceptance criterion through an alternative guard and explicitly document the premise-not-met decision with cross-referenced back-links so a future rebaser understands why no patch exists.

## Anti-Patterns
- Shipping a `skipif`-gated contract test without wiring it into the only workflow where it runs — a correct-but-never-invoked test gives false coverage confidence and a green CI that proves nothing.

## Suggested Skill Updates
- `testing-patterns`: add guidance that any test gated behind `skipif` (clone/host/resource-conditional) must be paired with (a) the runbook/workflow step that creates the condition and (b) a doc-assertion test that fails if the runbook stops invoking it.
