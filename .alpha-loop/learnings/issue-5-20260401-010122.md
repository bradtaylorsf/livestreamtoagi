## What Worked
- Comprehensive model registry with Decimal-based cost estimation covered all 9 project models accurately
- Fire-and-forget cost logging pattern (swallow DB errors, log warning) kept LLM calls resilient without sacrificing observability
- Retry logic with Retry-After header respect and exponential backoff handled 429/500/502/503 correctly across both streaming and non-streaming paths

## What Failed
- Streaming cost logging was placed after the `try/finally` block, causing silent cost data loss on early generator termination (fixed by reviewer moving it into `finally`)
- 3 retries took 3 attempts total (not 1 + 3 retries = 4) — naming mismatch with spec, though behavior is reasonable

## Patterns
- Place async generator cleanup logic (cost logging, tracing) inside `finally` blocks, not after them — `GeneratorExit` skips post-finally code
- Use `Decimal` for financial calculations from the start; avoid float-to-decimal conversion at boundaries
- Skip integration tests gracefully with `pytest.mark.skipif` when env vars are missing, rather than failing or mocking the external service

## Anti-Patterns
- Assuming async generator consumers will always exhaust the iterator — design for early termination as the common case
- Naming a constant `MAX_RETRIES` when it actually means `MAX_ATTEMPTS` — causes spec ambiguity
- Including unrelated file changes (`frontend/package-lock.json`) in feature branches — adds PR noise
