## What Worked
- Snapshot `created_at` fallback implemented at the API layer using `datetime.fromtimestamp(f.stat().st_mtime, tz=UTC)` when `snapshot_at` is missing — avoided a database migration since snapshot data lives in JSON files on disk.
- Fix applied symmetrically to both admin (`core/admin/simulation_routes.py`) and public (`core/public_routes.py`) endpoints, with paired tests (`test_snapshot_at_falls_back_to_file_mtime`, `test_snapshot_at_uses_persisted_value_when_present`) covering both paths.
- All 1971 backend tests + 335 frontend + 206 website tests passed on first run with zero retries.

## Patterns
- For "missing column" UI bugs where data lives in files, prefer file-stat fallback at the read path over a schema migration + backfill — simpler, no downtime, and the file mtime is authoritative.
- Pair admin and public route changes: when a field is exposed via two endpoints, fix and test both in the same patch to prevent drift.
- Test both branches of a fallback (persisted value present vs. absent) to lock in the precedence rule.

## Anti-Patterns
- The PR scope drifted well beyond #415: it added a scenario-launcher endpoint (`/scenarios`), `seed_file`/`max_cost` fields on `NewSimulationRequest`, refactored `_build_agent_profile`, and added `simulation_id`/`status` to public response models. None of this is required by the acceptance criteria — bundling unrelated changes makes review harder and bisection painful.
- `.gitignore` churn (adding `.venv` and `venv` without trailing slash when slashed variants already exist) is unrelated noise.
