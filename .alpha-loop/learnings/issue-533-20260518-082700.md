## Patterns
- For "fork + pin + reproducible install" issues: commit only the lockfile, git-ignore the working clone, and gate installs with `npm ci` + a HEAD-SHA hard assertion.

## Anti-Patterns
- Idempotent-reuse path runs `git fetch origin` without re-validating the existing clone's `origin` URL matches `$MINDCRAFT_REPO`; a stale clone pointed at a different remote won't auto-switch (mitigated by hard pin assertion + troubleshooting docs, but worth hardening).
