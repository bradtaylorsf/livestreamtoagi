## Patterns
- JSX text children do NOT interpret `\u` escape sequences — use `{"\u2014"}` or the literal `—` character. String literals in JS expressions are fine.
- When a data column is "empty," distinguish "not yet computable" (hide with threshold) from "genuinely null" (show em dash) rather than conflating them.

## Anti-Patterns
- Bundling unrelated refactors (scenario launcher, agent profile helper extraction) into a targeted bug-fix issue makes review harder and obscures the actual fix.
- Persisting default sentinel values (0.0, 0.5) without a flag distinguishing them from real measurements creates downstream ambiguity for non-UI consumers reading the table directly.
- Adding internal navigation links to routes that don't exist yet — either build the route or link to an external source (GitHub).
