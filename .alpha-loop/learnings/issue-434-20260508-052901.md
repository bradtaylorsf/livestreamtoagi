## Patterns
- State-machine claim pattern (`claim_for_render`/`claim_for_youtube`) using conditional UPDATE ... WHERE status IS NULL guarantees idempotent enqueue under concurrent triggers.
- Master env flag (`YOUTUBE_PUBLISH_ENABLED`) gating per-row opt-in (`publish_to_youtube`) gives ops a global kill-switch without touching individual sim rows.
- Mocking the YouTube client at the module boundary (rather than the `googleapiclient` HTTP layer) keeps test setup minimal and resilient to library changes.

## Anti-Patterns
- Synchronous `time.sleep` inside async functions (publish_simulation_youtube.py:239) — works in a one-shot subprocess but should use `await asyncio.sleep` to avoid blocking the event loop if the code is later reused.
- Brittle URL parsing via `rsplit('v=', 1)[-1]` for video ID extraction — only works for the canonical watch URL; would silently break on `youtu.be/ID` or `?v=ID&t=1s` query strings. Use `urllib.parse` instead.
