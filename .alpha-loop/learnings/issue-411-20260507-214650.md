## Patterns
- For "coming soon" stub pages, pair the empty-state copy with a small dedicated test file (`<page>-page.test.ts`) asserting the explanatory copy and CTA render — gives cheap regression coverage.
- Bundling related fixes (clips empty-state + adjacent admin/public-API changes) into one batch worked cleanly when the changes share a worktree and don't cross-couple test fixtures.

## Anti-Patterns
- Diff scope crept well beyond the issue (admin simulation routes, scenario listing, public API agent profile refactor, `.gitignore` edits) — for a UI-copy fix this is risky surface area; future runs should keep stub-page fixes scoped to the page and its test.
- Adding `.venv` and `venv` to `.gitignore` inside an issue PR is unrelated tooling drift; belongs in a separate chore commit.
