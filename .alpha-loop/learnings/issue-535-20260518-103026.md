## What Failed
- Review caught a real acceptance-criterion gap: the distinctness guard only checked always-distinct template tokens, not the resolved runtime env ids, so all-four-identical models passed silently and the troubleshooting doc claimed a warning that didn't exist (fixed: non-fatal `warn_runtime_model_ids()` advisory + doc correction + 3 regression tests).

## Patterns
- "Verify, don't patch": when a prior decision (E1-R3/#520) concludes native support, the issue's job is concrete two-instance proof + lock-step tests, not new code.

## Anti-Patterns
- Validating substitution *templates* instead of *resolved runtime values* — guards must assert on post-substitution env (LLM_A_CHAT etc.), since templates are distinct by construction and hide the degenerate case.
