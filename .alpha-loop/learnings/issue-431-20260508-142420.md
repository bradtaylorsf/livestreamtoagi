## Patterns
- Per-tab component split under `src/components/simulation/` with a thin page that maps `?tab=` → component keeps tab logic testable in isolation (each tab gets its own `__tests__` file).
- Owner-gated PATCH endpoints: validate UUID → fetch → check `submitted_by_user_id == user.id` → delegate partial update to repo with `None`-skipping kwargs.
- Empty-state-first chart components (Energy) — render guidance copy when the timeseries endpoint returns no rows, then swap in recharts once data arrives.

## Anti-Patterns
- Save buttons that 403 silently on anonymous-submitted sims: PATCH ownership check is correct, but the UI exposes editable controls without a read-only state for `submitted_by_user_id IS NULL`, leading to confusing inline "Unauthorized" errors.
- Calling the recall-counts endpoint a "memory evolution chart" in the issue when the backend exposes counts only — language drift between issue text and available data forced a click-through fallback rather than a true time series.
