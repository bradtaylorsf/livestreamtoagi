## Patterns
- When an endpoint hardcodes a scope constant (e.g. `LIVE_SIMULATION_ID`), check whether the underlying repo method already supports `None`/optional scoping before adding new parameters.
- For list endpoints that need both scoped and unscoped modes, branch the query construction explicitly rather than threading `Optional` through a single parameterized query — keeps SQL readable and arg positions stable.
- Test both the presence and absence of `simulation_id` in generated SQL via `mock_db.fetch.call_args[0][0]` substring checks.

## Anti-Patterns
- Hardcoding `LIVE_SIMULATION_ID` as a default filter on UUID-keyed lookups when collisions are impossible — silently breaks non-live access.
- Trusting `.gitignore` directory entries to cover symlinks pointing into the same path; verify with `git status` before commit.
- Leaving stale "must filter by simulation_id" assertions in place after inverting the behavior; replace them rather than deleting silently.
