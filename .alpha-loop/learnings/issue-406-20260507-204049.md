## Patterns
- Extract a `_build_*_profile` helper when an endpoint aggregates several repos with a shared scope param — keeps `simulation_id` plumbing consistent and testable.
- When fixing "stats card lies" bugs, add `simulation_id` to the response model (e.g. `ConversationSummary.simulation_id`) so the frontend can verify scope rather than trusting the count.
- Pair each scope-handling fix with a test that asserts *both* scoped and unscoped paths use the same scope across all displayed metrics.

## Anti-Patterns
- Don't hard-code `LIVE_SIMULATION_ID` for cost lookups while leaving other counts unscoped — produces the exact "$0.18 cost but 0 conversations" inconsistency this issue fixed.
- Don't ship a backfill script without flagging that it requires a manual run against prod for the verification step to pass.
- Inconsistent try/except wrapping between autonomous vs phased orchestrator paths (orchestrator.py:841-851 vs 588-613) — pre-existing but worth normalizing next time that file is touched.
