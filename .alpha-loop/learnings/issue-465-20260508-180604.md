## What Worked
- Synchronous URL parsing in `SimulationContext` seeds `activeSimulationId` from `/simulations/[id]/...` before first paint, so the picker shows the URL-derived sim immediately (verified by Playwright script under 3s API throttle — bare UUID renders, never the prior sim's name).
- `buildPickerLabel` stale-detail guard (`active.id === activeId`) prevents flashing the previous simulation's name when `activeDetail` lags behind a navigation; covered by a dedicated unit test.
- Back/forward navigation also avoids the old-label flash because the URL-derived id is the source of truth on every render.

## What Failed
- One test-fix retry was needed before the suite went green (specific failure not captured in artifacts).
- Width-only layout shift on hydration: button shrinks ~32px when bare UUID (capped at `max-w-[16rem]`) upgrades to `name · status`. Right edge is anchored so surrounding content is unaffected, but the strict "no layout shift" criterion is technically violated on the width axis.
- bfcache restore on back/forward skips the `/api/simulations*` fetch, so `activeDetail` stays stale and the picker remains pinned to the bare UUID (correct id, no friendly name). Out of strict scope for the AC.

## Patterns
- Derive client state synchronously from `usePathname()` / URL inside the provider's `useState` initializer to eliminate first-render flicker, instead of waiting for a `useEffect` to reconcile after mount.
- Guard label rendering with an explicit identity check (`detail.id === activeId`) so async detail fetches that resolve out of order can never paint a stale label.
- Hard-code a sentinel set (e.g. `SIMULATION_LIST_SENTINELS = {live, new, scenarios}`) to disambiguate sibling static routes from `[id]` segments when parsing the URL.

## Anti-Patterns
- Hard-coded sentinel sets drift silently as new static sub-routes are added (e.g. a future `/simulations/compare`) — a sibling route would be misread as a simulation id with no test failure to catch it.
- Both the root `SimulationProvider` (app layout) and the scoped provider (`/simulations/[id]/layout`) call `setCurrentSimulationId(pathSimId)` in their own effects; harmless thanks to the store's `memoryValue === id` dedupe, but the redundancy is easy to mistake for required behavior.
- Capping the trigger label width with `max-w-[16rem]` while letting the inner text length drive width creates an avoidable shrink-on-hydration; either pin a fixed min-width or skeleton the friendly portion to keep width stable.
