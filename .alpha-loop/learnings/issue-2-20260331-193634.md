## Patterns
- Pin Docker images to major version tags (`langfuse/langfuse:2`) instead of `latest` to avoid surprise breaking changes
- Use environment variable defaults with `${VAR:-default}` in compose files to allow per-developer overrides without `.env` churn
- Mount init SQL scripts as read-only (`:ro`) into `/docker-entrypoint-initdb.d/` for one-time bootstrap

## Anti-Patterns
- Using weak placeholder secrets as defaults — services with security requirements (Langfuse SALT) will reject them at runtime
- Changing default host ports (5432→5434) without documenting the change; breaks assumptions for other developers
- Three identical "fix: resolve verification failures" commit messages — messy history that obscures what each retry actually fixed

## Suggested Skill Updates
- Add a verification step that checks all HTTP-accessible services actually return 200, not just that the container is running
- Add a check for env var strength requirements when services have known minimums (e.g., Langfuse SALT length)
- After retry loops, squash or reword fix commits before opening the PR to keep history clean
