# Cost / Kill Audit

Issue: #594, E11-1. Scope: document the current cost and kill-switch mechanisms and the 24/7 gaps that feed E11-2 through E11-7.

## Summary

The current tree has a real simulation-level cost cap, a rolling aggregate spend cap for persistent runs, cost event logging, a hard per-agent hourly spend cap, admin kill-switch routes, and Management severity-5 kill behavior. It does not yet have spend/kill alerting or full kill propagation to Node bots and Minecraft/world-loop processes.

## Cost Path Details

1. LLM provider selection and pricing
   - OpenRouter is the default provider; local providers reuse the same client with `LOCAL_LLM_BASE_URL`, `LOCAL_LLM_MODEL`, and optional `LOCAL_LLM_MODEL_BUILDING` (`core/bootstrap.py:185`, `core/bootstrap.py:200`, `core/bootstrap.py:204`, `core/bootstrap.py:208`, `core/bootstrap.py:209`).
   - OpenRouter pricing is refreshed at bootstrap only for the OpenRouter provider (`core/bootstrap.py:284`, `core/bootstrap.py:285`, `core/bootstrap.py:286`), with fallback prices in `MODEL_REGISTRY` (`core/llm_client.py:93`, `core/llm_client.py:95`).
   - Local providers return zero estimated cost (`core/llm_client.py:387`, `core/llm_client.py:390`) but still log usage details through `_log_cost()`.

2. Authoritative per-simulation accounting
   - `CostRepo.add_cost()` is the central write path for `cost_events` (`core/repos/cost_repo.py:37`, `core/repos/cost_repo.py:39`).
   - `SimulationRepo.get_total_cost_from_events()` derives simulation totals from `cost_events` (`core/repos/simulation_repo.py:534`, `core/repos/simulation_repo.py:540`).
   - `_check_cost_limit()` and `_finalize()` both reconcile simulation totals from `cost_events` (`core/simulation/orchestrator.py:1139`, `core/simulation/orchestrator.py:1213`, `core/simulation/orchestrator.py:1215`).

3. Post-call and optional-cost paths
   - LLM calls always attempt to log after provider response; failures are retried four total attempts (`core/llm_client.py:437`, `core/llm_client.py:440`).
   - Web search and fetch have fixed estimated costs and only log when `cost_repo` is injected (`tools/web_tools.py:60`, `tools/web_tools.py:105`, `tools/web_tools.py:263`).
   - Journal image and PixelLab image generation log fixed estimates; PixelLab's estimate is explicitly amortized because the API is subscription based (`tools/journal_image_tool.py:201`, `core/world/pixellab_client.py:36`, `core/world/pixellab_client.py:237`).

## Kill Path Details

1. Operator kill path
   - The operator path is `POST /api/admin/kill` with `X-Kill-Switch-Key`; activation writes raw Redis key `kill_switch=active` for 14,400 seconds (`core/admin/kill_switch_routes.py:23`, `core/admin/kill_switch_routes.py:39`, `core/admin/kill_switch_routes.py:46`).
   - The route does not depend on general admin auth (`core/admin/__init__.py:36`, `core/admin/__init__.py:38`) and is mounted by `core/main.py` (`core/main.py:146`).

2. Python loop stop path
   - Autonomous simulations stop when `_terminated()` sees raw Redis `kill_switch == "active"` (`core/simulation/orchestrator.py:1074`, `core/simulation/orchestrator.py:1084`, `core/simulation/orchestrator.py:1085`).
   - Seeded simulation phases do not call `_terminated()`; their hard stop path today is cost limit or explicit `cancel()` (`core/simulation/orchestrator.py:727`, `core/simulation/orchestrator.py:728`, `core/simulation/orchestrator.py:1123`).

3. Management stop path
   - Severity 5 mutes the agent and sets the raw global kill switch with a 4-hour TTL (`core/management.py:204`, `core/management.py:205`, `core/management.py:206`).
   - E11-4 keeps per-agent mute keys on scoped Redis but uses an injected raw Redis client for the global kill key, matching the admin route and orchestrator read path.

4. Bridge / bot stop path
   - Current bridge kill behavior is limited to Alpha errand polling/completion (`core/bridge/server.py:440`, `core/bridge/server.py:445`).
   - The committed Node bridge client has fail-closed transport behavior, but no implemented `kill.status` subscription/polling path is present in the current service registry (`scripts/minecraft/fork-src/agent/bridge/python_bridge.js:31`; `core/bridge/contract.py:553`, `core/bridge/contract.py:554`).
